数据库三范式：
   1.原子性，字段值具有原子性，不可再分。
   2.唯一性，一个表必须有主键且唯一，非主键字段依赖主键。  
     （“一张表有多个主键”称为联合主键(复合主键)，就是用多个字段作为一张表的主键; 主键的作用：保证数据的唯一性和完整性，增加检索速度。）
   3.独立性，非主键字段之间不能相互依赖。
   
   
命名规范：
   1.表名，字段名必须是小写或数字，禁止数字开头。
   2.表名不使用复数名词。
   3.小数类型使用decimal禁止使用float
   4.索引命名：主键索引名"pk_字段名"  唯一索引名"uk_字段名"  普通索引名"idx_字段名"

   
 索引设计规范：
    1.单表索引控制在5个以内
	2.单索引字段不允许超过5个
	3.禁止在更新频繁，区分度不高的属性上建立索引
	4.复合索引，必须把区分度高的字段放在前面
	

 
















   
 
 PS：
   联合主键创建：
      1.方法一：
          Create Table 表名 ( 字段名1 Int Not Null,
                                  字段名2 nvarchar(13) Not Null Primary Key (字段名1, 字段名2),
                                  字段名3…
                                  字段名N… 
								);
								
      2.方法二：
	     建表完成后，更改字段：
	      ALTER TABLE 表名 ADD PRIMARY KEY(字段名1, 字段名2);
